MiTienda Web implements cookie-based authentication with role-based access control, allowing users to register, log in, and access protected features.

## Overview

The authentication system provides:
- **User registration** with email and password
- **Cookie-based login** using ASP.NET Core Authentication
- **Role-based authorization** for different user types
- **Claims-based identity** for accessing user information
- **Secure session management** with configurable persistence

## User Experience

<Steps>

<Warning>

</Warning>

### Login Flow

The login process validates credentials and creates an authentication cookie:

```csharp CuentaController.cs:18
[HttpPost]
public async Task <IActionResult> Login(LoginVM viewmodel)
{

    if (!ModelState.IsValid) return View(viewmodel);
    var found = await _usuarioService.Login(viewmodel);

    if (found.UsuarioId == 0)
    {
        ViewBag.MensajeExito = "No hay coincidencias";
        return View();


    }
    else
    {
        List<Claim>claims = new List<Claim>
        {
            new Claim(ClaimTypes.NameIdentifier, found.UsuarioId.ToString()),
            new Claim(ClaimTypes.Name, found.NombreCompleto),
            new Claim(ClaimTypes.Email, found.Correo),
            new Claim(ClaimTypes.Role, found.Rol)
        };

        var claimsIdentity = new ClaimsIdentity(
            claims, CookieAuthenticationDefaults.AuthenticationScheme);
        await HttpContext.SignInAsync(
            CookieAuthenticationDefaults.AuthenticationScheme,
            new ClaimsPrincipal(claimsIdentity),
            new AuthenticationProperties() {AllowRefresh= true, IsPersistent= true }
            );
        return RedirectToAction("Index", "Home");
    }
}
```

<Info>
  **Authentication Properties**:
  - `AllowRefresh = true`: Allows the authentication cookie to be refreshed
  - `IsPersistent = true`: Cookie persists across browser sessions ("Remember Me" behavior)
</Info>

### User Validation

The `UsuarioService` validates credentials against the database:

```csharp UsuarioService.cs:11
public async Task<UsuarioVM> Login(LoginVM loginVM)
{
    var conditions = new List<Expression<Func<Usuario, bool>>>()
    {
        x => x.Correo == loginVM.Correo,
        x => x.Contrasena == loginVM.Contrasena,
    };

    var found = await _usuarioRepository.GetByFilter(conditions: conditions.ToArray());

    var usuarioVM = new UsuarioVM();

    if (found != null)
    {
        usuarioVM.UsuarioId = found.UsuarioId;
        usuarioVM.NombreCompleto = found.NombreCompleto;
        usuarioVM.Correo = found.Correo;
        usuarioVM.Rol = found.Rol;
    }

    return usuarioVM;
}
```

### Registration Process

New users can register with validation for duplicate emails and password matching:

```csharp UsuarioService.cs:34
public async Task Registro(UsuarioVM usuarioVM)
{
    if(usuarioVM.Contrasena!= usuarioVM.RepiteContrasena)
        throw new InvalidOperationException("Las contraseñas no son iguales");

    var conditions = new List<Expression<Func<Usuario, bool>>>()
    {
        x => x.Correo == usuarioVM.Correo,
    };

    var foundCorreo = await _usuarioRepository.GetByFilter(conditions:conditions.ToArray());

    if (foundCorreo != null)
        throw new InvalidOperationException("El correo ya ha sido registrado");

    var entity = new Usuario()
    {
        NombreCompleto = usuarioVM.NombreCompleto,
        Correo = usuarioVM.Correo,
        Rol = usuarioVM.Rol,
        Contrasena = usuarioVM.Contrasena,
    }; 

    await _usuarioRepository.AddAsync(entity);
}
```

<Tabs>
  <Tab title="Validation 1">
    **Password Confirmation**: Ensures both password fields match before processing registration.
  </Tab>
  
  <Tab title="Validation 2">
    **Unique Email**: Checks that the email address isn't already registered in the system.
  </Tab>
  
  <Tab title="Error Handling">
    Validation failures throw `InvalidOperationException` with descriptive messages displayed to users.
  </Tab>
</Tabs>

## Claims-Based Identity

Once authenticated, user information is available via claims throughout the application:

```csharp
// Access user ID in any controller
var userId = HttpContext.User.FindFirst(ClaimTypes.NameIdentifier)!.Value;

// Check if user is authenticated
if (User.Identity.IsAuthenticated)
{
    // User-specific logic
}

// Access user role
var role = User.FindFirst(ClaimTypes.Role)?.Value;
```

<Note>
  Claims are stored in the authentication cookie and are available on every request without additional database queries.
</Note>

## Security Considerations

<Warning>
  **Production Recommendations**:
  
  1. **Password Hashing**: Replace plain-text passwords with proper hashing (BCrypt, Argon2)
  2. **HTTPS Only**: Enable `CookieSecure = CookieSecurePolicy.Always`
  3. **Anti-Forgery**: Add `[ValidateAntiForgeryToken]` to all POST actions
  4. **Account Lockout**: Implement lockout after failed login attempts
  5. **Password Strength**: Enforce password complexity requirements
  6. **Two-Factor Auth**: Consider adding 2FA for sensitive operations
</Warning>
