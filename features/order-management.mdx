The order management system handles the complete order lifecycle from cart checkout to order completion, including order creation, item tracking, and order history.

## Overview

Order management provides:
- **Seamless checkout** from shopping cart to completed order
- **Order persistence** in the database with full details
- **Line item tracking** for each product in an order
- **Order history** for users to view past purchases
- **Automatic total calculation** based on cart contents

public async Task<IActionResult> PagarAhora()
{

### Order Creation Service

The `OrdenService` handles the complex logic of converting cart items to order entities:

```csharp OrdenService.cs:11
public async Task AddAsync(List<ArticuloCarritoVM> carritoVM, int usuarioId)
{
    Orden orden = new Orden
    {
        FechaOrden = DateTime.Now,
        UsuarioId = usuarioId,
        TotalMonto = carritoVM.Sum(x => x.Precio * x.Quantity),
        ArticuloPedidos = carritoVM.Select(x => new ArticuloPedido
        {
            ProductoId = x.ProductId,
            Cantidad = x.Quantity,
            Precio = x.Precio
        }).ToList()
    };

    await _ordenRepository.AddAsync(orden);
}
```

<Tabs>
  <Tab title="Order Header">
    The `Orden` entity captures:
    - **FechaOrden**: Current timestamp when order is placed
    - **UsuarioId**: Links order to the authenticated user
    - **TotalMonto**: Calculated sum of all line items (price × quantity)
  </Tab>
  
  <Tab title="Order Lines">
    Each cart item becomes an `ArticuloPedido`:
    - **ProductoId**: Links to the product
    - **Cantidad**: Quantity ordered
    - **Precio**: Price at time of order (snapshot for historical accuracy)
  </Tab>
  
  <Tab title="Calculation">
    The LINQ expression calculates the total:
    ```csharp
    TotalMonto = carritoVM.Sum(x => x.Precio * x.Quantity)
    ```
    
    This automatically handles any number of cart items with any quantities.
  </Tab>
</Tabs>

### Order History

Users can retrieve their complete order history:

```csharp OrdenService.cs:29
public async Task<List<OrdenVM>> GetAllByUserAsync(int usuarioId)
{
    var ordenes = await _ordenRepository.GetAllWithDetailsAsync(usuarioId);
    
    var ordenesVM = ordenes.Select(x => new OrdenVM
    {
        FechaOrden = x.FechaOrden.ToString("dd/MM/yyyy"),
        MontoTotal = x.TotalMonto.ToString("C2"),
        ordenArticulos = x.ArticuloPedidos.Select(x => new OrdenArticuloVM
        {
            NombreArticulo = x.Producto.Nombre,
            Quantity = x.Cantidad,
            Precio = x.Precio.ToString("C")
        }).ToList()
    }).ToList();

    return ordenesVM;
}
```

<Info>
  **Formatted Output**:
  - Dates formatted as `dd/MM/yyyy` (e.g., "15/03/2026")
  - Currency formatted with `ToString("C2")` for totals and `ToString("C")` for items
  - Product names retrieved via navigation properties
</Info>

## Database Considerations

### Entity Framework Relationships

The order system relies on proper EF Core navigation properties:

```csharp
// In DbContext configuration
modelBuilder.Entity<Orden>()
    .HasMany(o => o.ArticuloPedidos)
    .WithOne(a => a.Orden)
    .HasForeignKey(a => a.OrdenId);

modelBuilder.Entity<Orden>()
    .HasOne(o => o.Usuario)
    .WithMany(u => u.Ordenes)
    .HasForeignKey(o => o.UsuarioId);
```

### Eager Loading

When retrieving order history, the repository should eagerly load related data:

```csharp
public async Task<IEnumerable<Orden>> GetAllWithDetailsAsync(int usuarioId)
{
    return await _context.Ordenes
        .Include(o => o.ArticuloPedidos)
            .ThenInclude(a => a.Producto)
        .Where(o => o.UsuarioId == usuarioId)
        .OrderByDescending(o => o.FechaOrden)
        .ToListAsync();
}
```

<Tip>
  Using `Include` and `ThenInclude` prevents N+1 query problems and ensures all order data is loaded efficiently in a single query.
</Tip>

## Data Integrity

### Why Store Price in Order Lines?

The `ArticuloPedido` entity includes a `Precio` field that duplicates the product price. This is intentional:

<CardGroup cols={2}>
  <Card title="Historical Accuracy" icon="file-check">
    Product prices may change over time. Storing the price at purchase time ensures order history remains accurate.
  </Card>
  
  <Card title="Reporting Integrity" icon="chart-line">
    Financial reports and order totals remain consistent even if current product prices differ.
  </Card>
</CardGroup>

<Warning>
  **Important**: If you only stored `ProductoId`, recalculating order totals later would use current prices, not the prices customers actually paid.
</Warning>

## Best Practices

1. **Transaction Handling**: Wrap order creation in a database transaction to ensure atomicity
2. **Stock Deduction**: Consider deducting product stock when orders are placed
3. **Order Status**: Add an order status field (Pending, Completed, Shipped, Cancelled)
4. **Email Confirmation**: Send order confirmation emails to customers
5. **Order Numbers**: Generate human-readable order numbers for customer reference
6. **Inventory Validation**: Verify stock availability before creating orders
