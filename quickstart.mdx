This guide will help you set up and run MiTienda Web quickly. Follow these steps to have your e-commerce platform running locally.

## Prerequisites

Before you begin, ensure you have:

- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0) installed
- [SQL Server](https://www.microsoft.com/sql-server/sql-server-downloads) (Express, Developer, or full version)
- A code editor (Visual Studio 2022, VS Code, or Rider)

<Info>
  You can verify your .NET installation by running `dotnet --version` in your terminal. It should show version 9.0 or higher.
</Info>

## Quick Setup

<Steps>
  <Step title="Clone the Repository">
    Clone the MiTienda Web repository to your local machine:

    ```bash
    git clone <repository-url>
    cd MiTiendaWeb_practica
    ```
  </Step>

  <Step title="Configure Database Connection">
    Open `MiTienda_practica/appsettings.json` and update the connection string with your SQL Server credentials:

    ```json appsettings.json
    {
      "Logging": {
        "LogLevel": {
          "Default": "Information",
          "Microsoft.AspNetCore": "Warning"
        }
      },
      "AllowedHosts": "*",
      "ConnectionStrings": {
        "CadenaSql": "Data Source=YOUR_SERVER;Database=DBTiendaWeb;User ID=YOUR_USER;Password=YOUR_PASSWORD;TrustServerCertificate=True;"
      }
    }
    ```

    <Note>
      For local SQL Server, the `Data Source` is typically `localhost` or `(localdb)\\mssqllocaldb` for LocalDB.
    </Note>
  </Step>

  <Step title="Apply Database Migrations">
    Navigate to the project directory and run Entity Framework migrations to create the database schema:

    ```bash
    cd MiTienda_practica
    dotnet ef database update
    ```

    This will create the `DBTiendaWeb` database with the following tables:
    - **Categorias** - Product categories (pre-seeded with Ropa, Electrónica, Hogar)
    - **Productos** - Products with pricing and inventory
    - **Usuarios** - User accounts with authentication
    - **Ordens** - Customer orders
    - **ArticuloPedidos** - Order line items

    <Tip>
      If you don't have Entity Framework tools installed, run:
      ```bash
      dotnet tool install --global dotnet-ef
      ```
    </Tip>
  </Step>

  <Step title="Run the Application">
    Start the application:

    ```bash
    dotnet run
    ```

    Or if you're using Visual Studio, press `F5` to run with debugging.

    The application will launch at:
    - HTTPS: `https://localhost:5001`
    - HTTP: `http://localhost:5000`

    <Info>
      By default, the application runs in Development mode with detailed error pages enabled.
    </Info>
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to `https://localhost:5001`

    You'll see the MiTienda Web home page. The application includes:
    - Product catalog browsing by category
    - Shopping cart functionality
    - User authentication and registration
    - Order management

    <Warning>
      On first launch, you may see a browser warning about the SSL certificate. This is normal for local development. Click "Advanced" and proceed to accept the self-signed certificate.
    </Warning>
  </Step>
</Steps>

## What's Configured?

The application comes pre-configured with:

### Database Context
Entity Framework Core with SQL Server provider for data persistence (see `/home/daytona/workspace/source/MiTienda_practica/Context/AppDbContext.cs`)

- **OrdenService** - Order processing

Cookie-based authentication with 30-minute session timeout and login redirect to `/Cuenta/Login` (Program.cs:34-41)

In-memory distributed cache with 30-minute idle timeout for shopping cart persistence (Program.cs:25-32)

## Troubleshooting

### Cannot connect to SQL Server

Ensure SQL Server is running and your connection string is correct. Test the connection:

```bash
sqlcmd -S YOUR_SERVER -U YOUR_USER -P YOUR_PASSWORD
```

### Migration errors

If migrations fail, try dropping the database and running migrations again:

```bash
dotnet ef database drop
dotnet ef database update
```

### Port already in use

If ports 5000/5001 are already in use, modify the launch settings in `Properties/launchSettings.json` or set different ports:

```bash
dotnet run --urls "https://localhost:7000;http://localhost:7001"
```
