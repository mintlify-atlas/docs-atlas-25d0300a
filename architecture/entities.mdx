## Overview

MiTienda Web uses **Entity Framework Core** with a code-first approach. All entities are defined as C# classes in the `Entities` folder and mapped to SQL Server tables through the `AppDbContext`.

## Core Entities

### Usuario (User)

Represents application users with authentication and role information.

```csharp
public class Usuario
{
    public int UsuarioId { get; set; }
    
    [Required]
    public string NombreCompleto { get; set; }
    
    [Required]
    public string Correo { get; set; }
    
    [Required]
    public string Contrasena { get; set; }
    
    [Required]
    public string Rol { get; set; }

    // Navigation property
    public ICollection<Orden> Ordenes { get; set; }
}
```

**Properties**:
- `UsuarioId`: Primary key, auto-generated
- `NombreCompleto`: Full name of the user
- `Correo`: Email address (used for login)
- `Contrasena`: Password (stored as plain text - should be hashed in production)
- `Rol`: User role (e.g., "Admin", "Cliente")
- `Ordenes`: Collection of orders placed by this user

Represents individual products within an order (many-to-many relationship).

```csharp

    public int ProductoId { get; set; }

    public decimal Precio { get; set; }

    public Producto? Producto { get; set; }
}
```

<Info>
**Location**: `/source/MiTienda_practica/Entities/ArticuloPedido.cs:3`
</Info>

**Properties**:
- `ArticuloPedidoId`: Primary key
- `OrdenId`: Foreign key to Orden
- `ProductoId`: Foreign key to Producto
- `Cantidad`: Quantity ordered
- `Precio`: Price at time of order (snapshot)
- `Orden`: Navigation property to order

<Note>
The `Precio` field stores the product price at the time of purchase, allowing historical orders to maintain accurate pricing even if product prices change.
</Note>

## Entity Configuration

All entity configurations are defined in `AppDbContext.OnModelCreating`:

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    base.OnModelCreating(modelBuilder);

    // Categoria configuration
    modelBuilder.Entity<Categoria>(e =>
    {
        e.HasKey("CategoriaId");
        e.Property("CategoriaId").ValueGeneratedOnAdd();
        e.HasData(
            new Categoria { CategoriaId = 1, Nombre = "Ropa" },
            new Categoria { CategoriaId = 2, Nombre = "Electrónica" },
            new Categoria { CategoriaId = 3, Nombre = "Hogar" }
        );
    });
    
    // Producto configuration
    modelBuilder.Entity<Producto>(e =>
    {
        e.HasKey("ProductoId");
        e.Property("ProductoId").ValueGeneratedOnAdd();
        e.Property("Precio").HasColumnType("decimal(10,2)");
        e.HasOne(e => e.Categoria)
         .WithMany(p => p.Productos)
         .HasForeignKey(e => e.CategoriaId)
         .OnDelete(DeleteBehavior.Restrict);
    });
    
    // Orden configuration
    modelBuilder.Entity<Orden>(e =>
    {
        e.HasKey("OrdenId");
        e.Property("OrdenId").ValueGeneratedOnAdd();
        e.Property("TotalMonto").HasColumnType("decimal(10,2)");
        e.HasOne(e => e.Usuario)
         .WithMany(p => p.Ordenes)
         .HasForeignKey(e => e.UsuarioId)
         .OnDelete(DeleteBehavior.Restrict);
    });
    
    // ArticuloPedido configuration
    modelBuilder.Entity<ArticuloPedido>(e =>
    {
        e.HasKey("ArticuloPedidoId");
        e.Property("ArticuloPedidoId").ValueGeneratedOnAdd();
        e.Property("Precio").HasColumnType("decimal(10,2)");
        e.HasOne(e => e.Orden)
         .WithMany(p => p.ArticuloPedidos)
         .HasForeignKey(e => e.OrdenId)
         .OnDelete(DeleteBehavior.Restrict);
        e.HasOne(e => e.Producto)
         .WithMany()
         .HasForeignKey(e => e.ProductoId)
         .OnDelete(DeleteBehavior.Restrict);
    });
}
```

## Relationship Types

<Tabs>
  <Tab title="One-to-Many">
    ### Usuario → Orden
    - One user can have many orders
    - Each order belongs to one user
    - Foreign key: `Orden.UsuarioId`
    
    ### Categoria → Producto
    - One category can have many products
    - Each product belongs to one category
    - Foreign key: `Producto.CategoriaId`
    
    ### Orden → ArticuloPedido
    - One order can have many line items
    - Each line item belongs to one order
    - Foreign key: `ArticuloPedido.OrdenId`
  </Tab>
  
  <Tab title="Many-to-Many">
    ### Producto ↔ Orden (through ArticuloPedido)
    - Products and Orders have a many-to-many relationship
    - `ArticuloPedido` serves as the join table
    - Includes additional data: `Cantidad`, `Precio`
    - This allows the same product to appear in multiple orders
    - And each order to contain multiple products
  </Tab>
</Tabs>

## Delete Behavior

All foreign key relationships use `DeleteBehavior.Restrict`:

<Warning>
Restricted delete means you **cannot delete** a parent record if child records exist. For example:
- Cannot delete a Usuario if they have Ordenes
- Cannot delete a Categoria if it has Productos
- Cannot delete a Producto if it's referenced in ArticuloPedidos

This prevents orphaned records and maintains referential integrity.
</Warning>

## DbContext Configuration

The `AppDbContext` exposes all entities as `DbSet<T>` properties:

```csharp
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options)
    {
    }
    
    public DbSet<Categoria> Categorias { get; set; }
    public DbSet<Producto> Productos { get; set; }
    public DbSet<Usuario> Usuarios { get; set; }
    public DbSet<Orden> Ordens { get; set; }
    public DbSet<ArticuloPedido> ArticuloPedidos { get; set; }
}
```

<Tip>
The DbContext is registered with dependency injection in `Program.cs:13` using SQL Server as the database provider.
</Tip>

## Best Practices Implemented

1. **Navigation Properties**: All relationships have navigation properties for easy traversal
2. **Data Annotations**: Required fields use `[Required]` attribute
3. **Nullable Reference Types**: Optional relationships use `?` operator
4. **Decimal Precision**: Money fields use `decimal(10,2)` for accurate currency representation
5. **Seed Data**: Initial categories ensure database has baseline data
6. **Fluent API**: Complex configurations use Fluent API in `OnModelCreating`
