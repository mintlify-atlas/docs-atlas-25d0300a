## Overview

The **Service Layer** contains all business logic for MiTienda Web. Services act as an intermediary between controllers and repositories, handling data transformation, validation, business rules, and coordination of complex operations.

## Service Architecture

```
Controllers
    ↓
┌─────────────────────────────────┐
│      Service Layer              │
│                                 │
│  ┌─────────────────────────┐   │
│  │  CategoriaService       │   │
│  │  ProductoService        │   │
│  │  OrdenService           │   │
│  │  UsuarioService         │   │
│  └─────────────────────────┘   │
│            ↓                    │
│  ┌─────────────────────────┐   │
│  │  Entity ↔ ViewModel     │   │
│  │  Transformation         │   │
│  └─────────────────────────┘   │
└─────────────┬───────────────────┘
              ↓
        Repositories
```

## Service Implementation Details

### CategoriaService

Manages product categories with basic CRUD operations.

```csharp
public class CategoriaService(GenericRepository<Categoria> _categoriaRepository)
{
    public async Task<IEnumerable<CategoriaVM>> GetAllAsync()
    {
        var categories = await _categoriaRepository.GetAllAsync();

        var categoriesVM = categories.Select(item =>
        new CategoriaVM
        {
            CategoriaId = item.CategoriaId,
            Nombre = item.Nombre
        }).ToList();

        return categoriesVM;
    }

    public async Task AddAsync(CategoriaVM viewModel)
    {
        var entity = new Categoria
        {
            Nombre = viewModel.Nombre
        };
        await _categoriaRepository.AddAsync(entity);
    }
    
    public async Task EditAsync(CategoriaVM viewModel)
    {
        var entity = new Categoria
        {
            CategoriaId = viewModel.CategoriaId,
            Nombre = viewModel.Nombre
        };
        await _categoriaRepository.EditAsync(entity);
    }

    public async Task DeleteAsync(int id)
    {
        var categoria = await _categoriaRepository.GetByIdAsync(id);
        await _categoriaRepository.DeleteAsync(categoria);
    }
}
```

### ProductoService

Handles product management with advanced features like image uploads and catalog filtering.

<Tabs>
  <Tab title="Get All Products">
    ```csharp
    public async Task<IEnumerable<ProductoVM>> GetAllAsync()
    {
        var productos = await _productoRepository.GetAllAsync(
            includes: new Expression<Func<Producto, object>>[]
            {
                x => x.Categoria!
            });

        var productosVM = productos.Select(item => new ProductoVM
        {
            ProductoId = item.ProductoId,
            Categoria = new CategoriaVM
            {
                CategoriaId = item.Categoria!.CategoriaId,
                Nombre = item.Categoria.Nombre
            },
            Nombre = item.Nombre,
            Descripcion = item.Descripcion,
            Precio = item.Precio,
            Stock = item.Stock,
            ImagenNombre = item.ImagenNombre
        }).ToList();

        return productosVM;
    }
    ```
    
    <Note>
    Uses eager loading with `includes` parameter to fetch related Categoria data in a single query.
    </Note>
  </Tab>
  
  <Tab title="Add Product with Image">
    ```csharp
    public async Task AddAsync(ProductoVM viewModel)
    {
        string? uniqueFileName = null;

        // 1. Handle image upload if present
        if (viewModel.ImagenArchivo != null)
        {
            string uploadsFolder = Path.Combine(
                webHostEnvironment.WebRootPath, "imagenes");

            // Create folder if it doesn't exist
            if (!Directory.Exists(uploadsFolder))
                Directory.CreateDirectory(uploadsFolder);

            uniqueFileName = Guid.NewGuid().ToString() + 
                Path.GetExtension(viewModel.ImagenArchivo.FileName);
            string filePath = Path.Combine(uploadsFolder, uniqueFileName);

            using (var fileStream = new FileStream(filePath, FileMode.Create))
            {
                await viewModel.ImagenArchivo.CopyToAsync(fileStream);
            }

            viewModel.ImagenNombre = uniqueFileName;
        }

        // 2. Create the product entity
        var entity = new Producto
        {
            CategoriaId = viewModel.Categoria.CategoriaId,
            Nombre = viewModel.Nombre,
            Descripcion = viewModel.Descripcion,
            Precio = viewModel.Precio,
            Stock = viewModel.Stock,
            ImagenNombre = viewModel.ImagenNombre
        };

        await _productoRepository.AddAsync(entity);
    }
    ```
    
    <Tip>
    Images are saved with GUID-based filenames to prevent naming conflicts and stored in `/wwwroot/imagenes/`.
    </Tip>
  </Tab>
  
  <Tab title="Edit Product">
    ```csharp
    public async Task EditAsync(ProductoVM viewModel)
    {
        var producto = await _productoRepository.GetByIdAsync(viewModel.ProductoId);

        if (viewModel.ImagenArchivo != null)
        {
            string uploadsFolder = Path.Combine(
                webHostEnvironment.WebRootPath, "imagenes");
            string uniqueFileName = Guid.NewGuid().ToString() + 
                Path.GetExtension(viewModel.ImagenArchivo.FileName);
            string filePath = Path.Combine(uploadsFolder, uniqueFileName);

            using (var fileStream = new FileStream(filePath, FileMode.Create))
                await viewModel.ImagenArchivo.CopyToAsync(fileStream);

            // Delete previous image if it existed
            if (!string.IsNullOrEmpty(producto.ImagenNombre))
            {
                var previousImage = producto.ImagenNombre;
                string deleteFilePath = Path.Combine(uploadsFolder, previousImage);

                if (File.Exists(deleteFilePath))
                    File.Delete(deleteFilePath);
            }

            viewModel.ImagenNombre = uniqueFileName;
        }
        else
        {
            viewModel.ImagenNombre = producto.ImagenNombre;
        }

        producto.CategoriaId = viewModel.Categoria.CategoriaId;
        producto.Nombre = viewModel.Nombre;
        producto.Descripcion = viewModel.Descripcion;
        producto.Precio = viewModel.Precio;
        producto.Stock = viewModel.Stock;
        producto.ImagenNombre = viewModel.ImagenNombre;

        await _productoRepository.EditAsync(producto);
    }
    ```
    
    <Warning>
    When updating a product image, the old image file is deleted from disk to prevent storage bloat.
    </Warning>
  </Tab>
  
  <Tab title="Catalog Filtering">
    ```csharp
    public async Task<IEnumerable<ProductoVM>> GetCatalogoAsync(
        int categoriaId = 0, 
        string? busqueda = null)
    {
        var conditions = new List<Expression<Func<Producto, bool>>>
        {
            x => x.Stock > 0
        }; 

        if (categoriaId != 0) 
            conditions.Add(x => x.CategoriaId == categoriaId);
            
        if (!string.IsNullOrEmpty(busqueda)) 
            conditions.Add(x => x.Nombre!.Contains(busqueda) || 
                               x.Descripcion!.Contains(busqueda));

        var productos = await _productoRepository.GetAllAsync(
            conditions: conditions.ToArray());

        var productosVM = productos.Select(item => new ProductoVM
        {
            ProductoId = item.ProductoId,
            Nombre = item.Nombre,
            Descripcion = item.Descripcion,
            Precio = item.Precio,
            Stock = item.Stock,
            ImagenNombre = item.ImagenNombre
        }).ToList();

        return productosVM;
    }
    ```
    
    <Info>
    Catalog filtering supports:
    - Filtering by category
    - Search by product name or description
    - Only shows products with stock > 0
    </Info>
  </Tab>

### OrdenService

Manages order creation and retrieval with order details.

<Tabs>
  <Tab title="Create Order">
    ```csharp
    public async Task AddAsync(List<ArticuloCarritoVM> carritoVM, int usuarioId)
    {
        Orden orden = new Orden
        {
            FechaOrden = DateTime.Now,
            UsuarioId = usuarioId,
            TotalMonto = carritoVM.Sum(x => x.Precio * x.Quantity),
            ArticuloPedidos = carritoVM.Select(x => new ArticuloPedido
            {
                ProductoId = x.ProductId,
                Cantidad = x.Quantity,
                Precio = x.Precio
            }).ToList()
        };

        await _ordenRepository.AddAsync(orden);
    }
    ```
    
    <Note>
    - Calculates total amount from cart items
    - Creates order and order items in a single operation
    - Uses OrdenRepository which handles stock reduction in a transaction
    </Note>
  </Tab>

    ```csharp
    public async Task<List<OrdenVM>> GetAllByUserAsync(int usuarioId)
    ```

### UsuarioService

Handles user authentication and registration.

<Tabs>
  <Tab title="Login">
    ```csharp
    public async Task<UsuarioVM> Login(LoginVM loginVM)
    {
        var conditions = new List<Expression<Func<Usuario, bool>>>()
        {
            x => x.Correo == loginVM.Correo,
            x => x.Contrasena == loginVM.Contrasena,
        };

        var found = await _usuarioRepository.GetByFilter(
            conditions: conditions.ToArray());

        var usuarioVM = new UsuarioVM();

        if (found != null)
        {
            usuarioVM.UsuarioId = found.UsuarioId;
            usuarioVM.NombreCompleto = found.NombreCompleto;
            usuarioVM.Correo = found.Correo;
            usuarioVM.Rol = found.Rol;
        }

        return usuarioVM;
    }
    ```
    
    <Warning>
    This implementation stores passwords in **plain text**, which is insecure. Production applications should use:
    - ASP.NET Core Identity
    - Password hashing (e.g., BCrypt, PBKDF2)
    - Salting
    </Warning>
  </Tab>
  
  <Tab title="Registration">
    ```csharp
    public async Task Registro(UsuarioVM usuarioVM)
    {
        // Validate passwords match
        if (usuarioVM.Contrasena != usuarioVM.RepiteContrasena)
            throw new InvalidOperationException("Las contraseñas no son iguales");

        // Check if email already exists
        var conditions = new List<Expression<Func<Usuario, bool>>>()
        {
            x => x.Correo == usuarioVM.Correo,
        };

        var foundCorreo = await _usuarioRepository.GetByFilter(
            conditions: conditions.ToArray());

        if (foundCorreo != null)
            throw new InvalidOperationException("El correo ya ha sido registrado");

        // Create new user
        var entity = new Usuario()
        {
            NombreCompleto = usuarioVM.NombreCompleto,
            Correo = usuarioVM.Correo,
            Rol = usuarioVM.Rol,
            Contrasena = usuarioVM.Contrasena,
        }; 

        await _usuarioRepository.AddAsync(entity);
    }
    ```
    
    <Note>
    **Business Rules Enforced**:
    - Password and confirmation must match
    - Email must be unique
    - All required fields must be provided
    </Note>
  </Tab>
</Tabs>

## Service Patterns & Best Practices

### 1. Primary Constructor Injection

All services use C# 12 primary constructor syntax for clean dependency injection:

```csharp
public class ProductoService(
    GenericRepository<Categoria> _categoriaRepository,
    GenericRepository<Producto> _productoRepository,
    IWebHostEnvironment webHostEnvironment
)
{
    // Dependencies are automatically available as fields
}
```

### 2. Entity ↔ ViewModel Transformation

Services handle all transformations between entities and view models:

```csharp
// Entity → ViewModel
var productoVM = new ProductoVM
{
    ProductoId = producto.ProductoId,
    Nombre = producto.Nombre,
    Precio = producto.Precio,
    // ... other properties
};

// ViewModel → Entity
var entity = new Producto
{
    Nombre = viewModel.Nombre,
    Precio = viewModel.Precio,
    // ... other properties
};
```

### 3. Business Logic Validation

Services enforce business rules before delegating to repositories:

```csharp
// Password validation
if (usuarioVM.Contrasena != usuarioVM.RepiteContrasena)
    throw new InvalidOperationException("Las contraseñas no son iguales");

// Duplicate email check
var foundCorreo = await _usuarioRepository.GetByFilter(conditions);
if (foundCorreo != null)
    throw new InvalidOperationException("El correo ya ha sido registrado");
```

### 4. Complex Operations Coordination

Services coordinate multiple repository operations:

```csharp
// ProductoService fetches product and categories
var producto = await _productoRepository.GetByIdAsync(id);
var categorias = await _categoriaRepository.GetAllAsync();

// Combines data for view model
productoVM.Categorias = categorias.Select(item => new SelectListItem
{
    Value = item.CategoriaId.ToString(),
    Text = item.Nombre
}).ToList();
```

### 5. File System Operations

ProductoService handles file uploads and deletions:

```csharp
// Save image
string uploadsFolder = Path.Combine(webHostEnvironment.WebRootPath, "imagenes");
if (!Directory.Exists(uploadsFolder))
    Directory.CreateDirectory(uploadsFolder);

uniqueFileName = Guid.NewGuid().ToString() + 
    Path.GetExtension(viewModel.ImagenArchivo.FileName);

// Delete old image
if (File.Exists(deleteFilePath))
    File.Delete(deleteFilePath);
```

## Service Registration

All services are registered with **scoped lifetime** in `Program.cs`:

```csharp
builder.Services.AddScoped<CategoriaService>();
builder.Services.AddScoped<ProductoService>();
builder.Services.AddScoped<OrdenService>();
builder.Services.AddScoped<UsuarioService>();
```

<Info>
**Scoped lifetime** means:
- One instance per HTTP request
- Shared across all components in the same request
- Disposed at the end of the request
</Info>

## Benefits of the Service Layer

<Steps>
  <Step title="Separation of Concerns">
    Controllers focus on HTTP concerns, services handle business logic
  </Step>
  
  <Step title="Reusability">
    Business logic can be reused across multiple controllers or even APIs
  </Step>
  
  <Step title="Testability">
    Services can be unit tested independently of controllers and repositories
  </Step>
  
  <Step title="Maintainability">
    Business rules are centralized and easy to locate and modify
  </Step>
</Steps>
