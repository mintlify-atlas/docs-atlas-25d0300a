## Prerequisites

Before setting up MiTienda Web, ensure you have the following installed on your development machine:

<Steps>
  <Step title="Install .NET 9 SDK">
    Download and install the [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0) for your operating system.

    Verify installation:
    ```bash
    dotnet --version
    ```

    You should see version `9.0.x` or higher.
  </Step>

  <Step title="Install SQL Server">
    Install one of the following:

    - **SQL Server 2019 or later** (Windows)
    - **SQL Server on Linux**
    - **SQL Server Express** (free edition)
    - **Azure SQL Database** (cloud option)

    <Tip>
      For local development, SQL Server Express or the Developer edition are recommended.
    </Tip>
  </Step>

  <Step title="Install an IDE">
    Choose your preferred development environment:

    <CardGroup cols={2}>
      <Card title="Visual Studio 2022" icon="microsoft">
        Full-featured IDE with excellent ASP.NET Core support
        - Community Edition (free)
        - Professional or Enterprise
      </Card>
      <Card title="Visual Studio Code" icon="code">
        Lightweight editor with C# extension
        - Install C# Dev Kit extension
        - Install C# extension
      </Card>
      <Card title="JetBrains Rider" icon="rocket">
        Cross-platform .NET IDE
        - Commercial license required
        - 30-day free trial
      </Card>
      <Card title="Visual Studio for Mac" icon="apple">
        macOS-native IDE
        - Good ASP.NET Core support
        - Being deprecated in 2024
      </Card>
    </CardGroup>
  </Step>

  <Step title="Install Entity Framework Core Tools">
    Install the EF Core command-line tools globally:

    ```bash
    dotnet tool install --global dotnet-ef
    ```

    Verify installation:
    ```bash
    dotnet ef --version
    ```

    <Note>
      The project already includes `Microsoft.EntityFrameworkCore.Tools` (v9.0.7) as a package reference, which provides design-time support.
    </Note>
  </Step>
</Steps>

## Clone the Repository

Clone the MiTienda Web repository to your local machine:

```bash
git clone <repository-url>
cd MiTienda_practica
```

## Project Structure

The project follows the standard ASP.NET Core MVC structure:

```
MiTienda_practica/
├── Context/              # Database context
│   └── AppDbContext.cs
├── Controllers/          # MVC controllers
├── Entities/            # Domain models
├── Migrations/          # EF Core migrations
├── Repositories/        # Data access layer
├── Services/            # Business logic layer
├── Views/              # Razor views
├── wwwroot/            # Static files
│   └── imagenes/       # Product images
├── appsettings.json    # Configuration
└── Program.cs          # Application entry point
```

## Restore Dependencies

Restore all NuGet packages:

```bash
dotnet restore
```

<Info>
  The project uses the following key packages:
  - **Microsoft.EntityFrameworkCore.SqlServer** (9.0.7)
  - **Microsoft.EntityFrameworkCore.Tools** (9.0.7)
  - **Microsoft.VisualStudio.Web.CodeGeneration.Design** (9.0.11)
</Info>

## Build the Project

Build the solution to verify everything is set up correctly:

```bash
dotnet build
```

If the build succeeds, you're ready to configure the database connection.

## IDE-Specific Setup

### Visual Studio 2022

1. Open `MiTienda_practica.sln` (or the `.csproj` file)
2. Set `MiTienda_practica` as the startup project
3. Configure the database connection (see [Configuration](/development/configuration))
4. Press `F5` to run with debugging or `Ctrl+F5` to run without debugging

### Visual Studio Code

1. Open the project folder in VS Code
2. Install recommended extensions when prompted:
   - C# Dev Kit
   - C# (powered by OmniSharp)
3. Open the terminal (`Ctrl+~`)
4. Configure the database connection (see [Configuration](/development/configuration))
5. Run the project:
   ```bash
   dotnet run
   ```

### JetBrains Rider

1. Open the project folder or `.csproj` file
2. Rider will automatically detect the project structure
3. Configure the database connection (see [Configuration](/development/configuration))
4. Click the Run button or press `Shift+F10`

## Troubleshooting

### .NET SDK Not Found

If you see "The current .NET SDK does not support targeting .NET 9.0", ensure you have .NET 9 SDK installed:

```bash
dotnet --list-sdks
```

### NuGet Restore Fails

Clear the NuGet cache and restore again:

```bash
dotnet nuget locals all --clear
dotnet restore
```

### Port Already in Use

If the default port (5000/5001) is already in use, modify `launchSettings.json` or specify a different port:

```bash
dotnet run --urls "https://localhost:5002;http://localhost:5003"
```
